# Repository Description

This repository contains markdown content that can be used for Documentation as Code (DaC) during proposal processes where there is a Coding Challenge (CC) requirement. 

## Mapping of Content

- README : Core read me template to be used to describe the entire repository and it's relathionship to the CC solution. 
- SOLUTION : Solution overview that is more technical. !!!NOTE!!! For some CC's this file may be requested to be titled ARCHITECTURE. Other times both SOLUTION and ARCHITECTURE may be requested and in these events SOLUTION will be a complete overview of the solution and ARCHITECTURE will purely focus on technical aspects.
- TEAM : Describes the team that worked on the CC solution and the associated organizations reponding to the RFP.
- PRODUCT_VISION : Overview of the CC solution that outlines goals, purpose, mission alignment, and value.
- UCD : Overview of design processes and artifacts created and used during the CC.
- AGILE : Describes Agile processes, frameworks, and artifacts used during the CC.
- CICD : Details concerning DevSecOps process used and the solution components that enable Continuous Integration / Continuous Delivery.
- QUALITY : Overview of all aspects of end-to-end Quality to include how automation and data support related processes.
- RTM : Mapping of CC requirements to solution componentes within the repository.

!!!NOTE!!! It is appropriate that a README file be put into the core repository for the CC. Requirements as to the content of this file may be express and should be evaluated per what is recieved with RFP and CC instruction. Additionally, each repository must contain a README as is appropriate, however, there may also be limitation of what exactly the per-repository README's may contain.
